Structure du Projet Python et Bibliothèques (Mis à jour)

Le projet sera divisé en deux scripts distincts pour répondre à votre besoin de sécurité : un pour vous (chiffrement) et un pour vos destinataires (déchiffrement).

1. Structure des Fichiers

Voici la nouvelle structure :

Projet_Chiffrement/
│
├── 🔒 encrypt.py    <-- L'outil de chiffrement (pour vous uniquement)
│
└── 🔑 decrypt.py    <-- L'outil de déchiffrement (à distribuer)


encrypt.py contiendra la logique pour :

Demander le fichier PDF d'origine.

Demander le mot de passe (ex: AstierAuRapport!).

Demander le texte du filigrane (Watermark) (ex: Pour Astier).

(MIS À JOUR) Créer une version temporaire du PDF qui inclut :

Le filigrane (Watermark) ajouté sur chaque page.

Les permissions PDF définies sur "lecture seule" (interdiction de modification, de copie et d'impression).

Chiffrer ce PDF temporaire sécurisé avec le mot de passe (en utilisant la cryptographie AES).

Supprimer le PDF temporaire.

decrypt.py contiendra uniquement la logique pour :

Demander le fichier chiffré (.enc).

Demander le mot de passe.

Déchiffrer le fichier. (Le PDF résultant sera en lecture seule et portera le filigrane).

2. Bibliothèques Python Requises

La liste s'allonge d'une bibliothèque pour gérer les PDF.

A. Bibliothèques Externes (à installer)

Vous devrez installer ces deux bibliothèques :

cryptography

Pourquoi ? Pour le chiffrement sécurisé (AES-GCM, PBKDF2). C'est le cœur de notre sécurité.

Installation : pip install cryptography

PyPDF2

Pourquoi ? C'est la bibliothèque qui nous permettra de lire votre PDF, d'y ajouter un "Watermark" (filigrane) ET de définir les autorisations de sécurité (lecture seule) avant de le chiffrer.

Installation : pip install PyPDF2

B. Modules Intégrés (inclus dans Python)

Ces modules seront utilisés dans les deux scripts (encrypt.py et decrypt.py).

os

Pourquoi ? Pour générer le "Sel" et le "Nonce" aléatoires.

getpass

Pourquoi ? Pour demander les mots de passe de manière sécurisée (sans les afficher).

argparse

Pourquoi ? Pour gérer les arguments de la ligne de commande (ex: python decrypt.py fichier.enc sortie.pdf).

sys

Pourquoi ? Pour gérer les messages d'erreur et quitter proprement le script.
